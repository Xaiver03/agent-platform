## 🎯 文档编写规范

### 格式要求
- 使用 Markdown 格式
- 文件名使用小写字母和连字符
- 包含适当的emoji图标增强可读性
- 保持结构清晰，使用合适的标题层级

### 内容要求
- 简洁明了，重点突出
- 包含实用的代码示例
- 及时更新，确保准确性
- 支持中英文双语（优先中文）

### 维护原则
- 每次功能更新后及时更新相关文档
- 定期检查链接有效性
- 收集用户反馈并改进文档质量
